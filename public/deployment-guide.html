🚀 دليل النشر السريع

اتبع هذه الخطوات لنشر تطبيق Expense Tracker

1 إعداد MongoDB Atlas

أولاً، سنقوم بإنشاء قاعدة بيانات مجانية على MongoDB Atlas:

  • سجل في MongoDB Atlas
  • أنشئ Cluster مجاني (M0)
  • أنشئ Database User مع اسم مستخدم وكلمة مرور
  • في Network Access، اختر "Allow Access from Anywhere"
  • احصل على Connection String وسيكون شكله:
mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/expense-tracker
⚠️ مهم: احتفظ بـ Connection String في مكان آمن!

2 رفع الكود على GitHub

افتح Terminal في مجلد المشروع وقم بتنفيذ:

git init
git add .
git commit -m "Ready for deployment"
git branch -M main
git remote add origin <YOUR-GITHUB-REPO-URL>
git push -u origin main

3 نشر Backend على Render

الآن سنقوم بنشر الـ Backend:

  • سجل في Render.com باستخدام GitHub
  • اضغط "New +" ثم "Web Service"
  • اختر المستودع الخاص بك
  • أدخل الإعدادات التالية:
Build Command: npm install
Start Command: npm start
Instance Type: Free

في Environment Variables أضف:

MONGO_URI = mongodb+srv://your-connection-string
NODE_ENV = production
✅ بعد النشر ستحصل على رابط مثل: https://expense-tracker-api.onrender.com

4 تعديل Frontend

⚠️ خطوة مهمة جداً قبل نشر الـ Frontend!

افتح ملف public/script.js وابحث عن السطر:

const API_BASE_URL = '/api/expenses';

واستبدله بـ:

const API_BASE_URL = 'https://expense-tracker-api.onrender.com/api/expenses';

أيضاً ابحث عن '/api/budgets' واستبدلها بـ:

'https://expense-tracker-api.onrender.com/api/budgets'
ملاحظة: استخدم الرابط الفعلي من Render في الخطوة السابقة

5 نشر Frontend على Netlify

الآن سنقوم بنشر واجهة المستخدم:

  • اذهب إلى Netlify.com
  • اضغط "Add new site" → "Deploy manually"
  • اسحب مجلد public بالكامل إلى الصفحة
  • انتظر حتى ينتهي الرفع
🎉 مبروك! تطبيقك الآن منشور على الإنترنت!

6 اختبار التطبيق

تأكد من أن كل شيء يعمل:

  • ✅ إضافة مصروف جديد
  • ✅ عرض المصروفات
  • ✅ حذف مصروف
  • ✅ تعيين ميزانية
  • ✅ عرض التقارير الشهرية

⚠️ مشاكل شائعة وحلولها:

  • CORS Error: تأكد من إضافة CORS في app.js (تم بالفعل ✅)
  • لا يمكن الاتصال بقاعدة البيانات: تحقق من صحة MONGO_URI
  • API لا يعمل: افتح Logs في Render للتحقق من الأخطاء
  • Frontend لا يتصل بـ Backend: تأكد من تعديل API_BASE_URL
